# Awesome Voice Input Methods

A curated list of voice keyboards, AI dictation apps, and open-source speech-to-text tools.

This repository focuses on tools that help you speak instead of type, then turn raw speech into usable text for writing, chatting, coding, and daily work.

Chinese version: [README.zh-CN.md](./README.zh-CN.md)

## Open-Source Projects

| Name | Platforms | Highlights | Links | Notes |
| --- | --- | --- | --- | --- |
| TypeNo | macOS | Privacy-first local voice input that transcribes quickly and pastes into the active app | [GitHub](https://github.com/marswaveai/TypeNo) | GPL-3.0 |
| type4me | macOS | Local and cloud ASR, prompt-based processing modes, local history, and export support | [GitHub](https://github.com/joewongjc/type4me) | MIT |
| Airtype | macOS | Menu bar voice-to-text app with pluggable transcription providers and optional LLM cleanup | [GitHub](https://github.com/sugarforever/airtype) | MIT |
| Koe | macOS | Background-first hotkey workflow with plain-text config, dictionary files, and lightweight native design | [GitHub](https://github.com/missuo/koe) | MIT |
| Ghost Pepper | macOS | Fully local hold-to-talk speech-to-text using WhisperKit and local LLM cleanup | [GitHub](https://github.com/matthartman/ghost-pepper) | MIT; English-only, not suitable for non-English regions |
| Voxt | macOS | Menu bar voice input & translation app with hold-to-talk, App Branch context rules, meeting transcription, personal dictionary, and rich local/cloud ASR+LLM provider support | [GitHub](https://github.com/hehehai/voxt) | Apache-2.0 |
| Handless | macOS | Open-source speech-to-text with local/cloud providers, custom prompt templates, and multi-language support | [GitHub](https://github.com/ElwinLiu/handless) | MIT |
| Coe | Linux (GNOME/Wayland) | Zero-GUI Linux dictation tool inspired by Koe, built for hotkey-driven voice input | [GitHub](https://github.com/quailyquaily/coe) | Useful if you want a Linux-first option |
